Slack-related utilities.

Methods

<static> convertProperties(obj [, deep])

Converts slack properties into camelCase JS ones using a cloned object. This method does not take circular references into account since they should not exist in slack's responses.

Parameters:
Name Type Argument Default Description
obj T

The object to convert.
deep boolean <optional>
 false

Whether the conversion should be deep, i.e. if it should enter further objects to convert their properties.
Returns:
Type
T
Example
// Returns {helloWorld: 'hi'}
SlackUtil.convertProperties({
 hello_world: 'hi!'
});

<async, static> getPages(method [, args], getData [, pageLimit] [, singlePageLimit] [, combine])

Get all pages for a slack api call. This is useful for anything that requires the use of pagination, for instance the conversations.list method call. Pass arguments inside an object, so they are named.

Parameters:
Name Type Argument Default Description
method function

The api function to call. Pass a reference to the client.api.methods{your method}
args object <optional>
 {}

Arguments to use when calling this method for the first time (consecutive calls don't require args since the cursor stores it)
getData function

A function which accepts slack data and returns the data to add to the pages returned
pageLimit number <optional>
 Number.MAX_SAFE_INTEGER

The maximum number of pages to retrieve. This could result in a lot of calls if you're in a large organization.
singlePageLimit number <optional>
 100

The maximum number of results for one page.
combine boolean <optional>
 true

Whether or not to "flatten" all pages into a single array before returning them. If this is false, each array member will be a page of data, and if it is true all elements from each page will be in the same array level.
Returns:
Type
Promise.<Array>
